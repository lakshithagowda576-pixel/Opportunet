#### 2. The "Automated HR" Logic
For the email feature, you can use a library like **Nodemailer** (for real emails) or simply create a frontend "Simulation" where a `setTimeout` function triggers a "New Message" notification from the HR dummy email 24 hours after the user clicks "Send."

#### 3. State Management
Since it is a single-page application with many moving parts (progress bars, different modules), using **React Context API** or **Redux** will be vital to ensure that when a user applies in the "IT Module," the "Applicant Dashboard" updates instantly.
